# DEMP Frontend User Login & Password Reset Deep Dive

This document provides a detailed, step-by-step guide and technical overview of the user login and password reset processes in the Digital Event Management Platform (DEMP) frontend application. It covers user experience, validation, security, troubleshooting, and backend integration.

---

## 1. Login Flow Overview
- **Access:** Click the "Login" button on the homepage or navigation bar.
- **Form Fields:** Email and password.
- **Submission:** On submit, the frontend validates input and sends a login request to the backend API.
- **Success:** User is redirected to their dashboard.
- **Failure:** Clear error messages are displayed (e.g., "Invalid credentials").

---

## 2. Step-by-Step Login Guide
1. Click "Login" on the homepage or navigation bar.
2. Enter your registered email and password.
3. Click "Login" to submit the form.
4. If credentials are correct, you are redirected to your dashboard.
5. If credentials are incorrect, an error message is shown.
6. If your account is not activated, you are prompted to check your email for the activation link.

---

## 3. Password Reset Flow
- **Forgot Password:**
  - Click the "Forgot Password" link on the login page.
  - Enter your registered email address.
  - Submit the form to receive a password reset email.
  - Open the email and click the reset link.
  - Enter a new password (must meet security requirements).
  - Submit to complete the reset and return to the login page.

---

## 4. Validation & Error Handling
- **Frontend Validation:**
  - Checks for required fields and valid email format.
  - Password must meet minimum length and complexity.
- **Backend Validation:**
  - Verifies credentials, account status, and password policy.
  - Returns error messages for invalid login or reset attempts.
- **Error Feedback:**
  - Clear, actionable error messages (e.g., "Email not registered", "Password too short").

---

## 5. Security Considerations
- **HTTPS:** All login and password reset data is sent over encrypted connections.
- **Password Handling:** Passwords are never stored or logged in the frontend; sent securely to backend for verification.
- **Account Lockout:** Backend may lock accounts after repeated failed attempts.
- **Token Expiry:** Password reset links expire after a set period for security.

---

## 6. Integration with Backend
- **Login API Endpoint:** `POST /api/users/login`
- **Password Reset Request Endpoint:** `POST /api/users/forgot-password`
- **Password Reset Completion Endpoint:** `POST /api/users/reset-password`
- **Payload Example (Login):**
  ```json
  {
    "email": "jane@example.com",
    "password": "StrongP@ssw0rd!"
  }
  ```
- **Payload Example (Password Reset Request):**
  ```json
  {
    "email": "jane@example.com"
  }
  ```
- **Payload Example (Password Reset Completion):**
  ```json
  {
    "token": "reset_token",
    "newPassword": "NewStrongP@ssw0rd!"
  }
  ```

---

## 7. Troubleshooting & Support
- **Login Issues:**
  - Double-check your email and password.
  - Use the "Forgot Password" link if needed.
  - Ensure your account is activated.
- **Password Reset Issues:**
  - Check spam/junk folder for reset email.
  - Ensure the reset link is not expired.
  - Contact support if you cannot reset your password.

---

## 8. Best Practices
- Use strong, unique passwords for your account.
- Do not share your password with anyone.
- Log out after each session, especially on shared devices.
